# Qor Gate Chain

**Version**: 1 (Phase 3)
**Enforcement**: Advisory (soft). Missing/invalid prior artifact surfaces a prompt; user override logs a `gate_override` event (severity 1) in `docs/PROCESS_SHADOW_GENOME.md`.

## Sequence

```
research -> plan -> audit -> implement -> substantiate -> validate -> (remediate?)
```

`remediate` is conditional â€” triggered by threshold breach in the Process Shadow Genome, or by explicit `/qor-remediate` invocation. Not part of the strict forward chain.

## Artifact locations

Runtime: `.qor/gates/<session_id>/<phase>.json`

`session_id` format: `<YYYY-MM-DDTHHMM>-<6hex>` (e.g. `2026-04-15T1743-a3f9c2`). No colons â€” safe as a directory name on Windows.

Generated by `qor/scripts/session.py get_or_create`. Cached in `.qor/current_session` file, regenerated if older than 24h.

## Per-phase reads and writes

| Phase | Reads prior | Writes | Schema |
|---|---|---|---|
| research | (none â€” chain start) | `research.json` | `qor/gates/schema/research.schema.json` |
| plan | `research.json` | `plan.json` | `plan.schema.json` |
| audit | `plan.json` | `audit.json` | `audit.schema.json` |
| implement | `audit.json` (must be PASS) | `implement.json` | `implement.schema.json` |
| substantiate | `implement.json` | `substantiate.json` | `substantiate.schema.json` |
| validate | `substantiate.json` | `validate.json` | `validate.schema.json` |
| remediate | `docs/PROCESS_SHADOW_GENOME.md` | `remediate.json` | `remediate.schema.json` |

## Skill integration (Phase 52 wiring)

Skills add to their YAML frontmatter:

```yaml
phase: plan
gate_reads: research
gate_writes: plan
```

At startup, skills call:

```python
from gate_chain import check_prior_artifact, emit_gate_override
result = check_prior_artifact(current_phase="plan")
if not result.found:
    # prompt user; if override, emit event
    emit_gate_override(current_phase="plan", prior_phase="research",
                       reason="...", session_id=session_id)
```

Skills write their output artifact near end-of-protocol:

```python
from validate_gate_artifact import write_artifact
write_artifact(phase="plan", data={...})
```

## Override semantics

Override is permitted for any gate. Each override:
- Logs a `gate_override` event (severity 1) with `details` capturing reason, prior_phase, current_phase
- Counts toward the Process Shadow Genome threshold (10 severity points -> issue creation)
- Is visible in `docs/PROCESS_SHADOW_GENOME.md` JSONL and via `qor/scripts/check_shadow_threshold.py`

Repeated overrides on the same gate indicate either a process defect (gate too strict) or a systemic shortfall (skill not producing artifacts). `/qor-remediate` evaluates patterns.

## Non-forward paths

- **Remediate loop**: `remediate` consumes the shadow log directly, not a prior gate artifact. It can be invoked after any phase.
- **Audit VETO**: if `audit.json.verdict == "VETO"`, `implement` should not proceed. Skill-layer enforcement (Phase 52 wiring): `QOR_SKILL_ACTIVE` env-var provenance binding in `gate_chain.write_gate_artifact` + `gate_chain_completeness.check()` invoked from `/qor-substantiate` Step 7.8 + pre-merge CI gate. Bypass surface closed.
- **Debug interrupt**: if regression/hallucination/degradation is detected during `implement` or `substantiate`, `/qor-debug` is the legal next action (not `implement` again). This is a skill-level rule, not a gate schema rule.
